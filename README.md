## 📖 About the Project

**AcademicFlow** is an advanced academic planner engine tailored for university students and advisors. It solves the complex combinatorial problem of scheduling courses while strictly adhering to prerequisite rules, credit limits, and optimal graduation timelines. 

Instead of relying on basic greedy approaches that often fail in deep prerequisite trees, this engine utilizes a highly optimized **A* (A-Star) Search Algorithm** over a **Directed Acyclic Graph (DAG)**. It explores state spaces (combinations of passed courses) using heuristic functions and top-K pruning techniques to avoid exponential memory explosions, guaranteeing near-instant study plans even for massive curriculums across different academic programs.

### ✨ Key Features
- **Intelligent Auto-Generation:** Calculates the fastest path to graduation without exceeding maximum credit loads per semester.
- **Interactive Curriculum Map:** A dynamic, drag-and-drop DAG visualization (powered by React Flow) that allows users to mark courses as completed and instantly recalculates the entire progress.
- **Critical Path Tracking:** Highlights bottleneck courses dynamically, warning students of subjects that could delay their graduation.
- **Multi-Program Support:** Filters courses by `program_id`, allowing a single backend to gracefully handle multiple degrees and pensums seamlessly.
- **Real-Time GPA & Progress Dashboard:** Features interactive Recharts analytics tracking credit pacing and GPA progression over time.

## 🚀 Getting Started

The project is decoupled into two main environments: the `backend` (FastAPI) and the `frontend` (React + Vite). You need to run both to get the full application working.

### 📋 Prerequisites
- **Python 3.10+** (For the Backend API)
- **Node.js 18+** & **pnpm** (For the Frontend application)

### 1️⃣ Setting up the Backend
The backend manages the algorithms, graph logic, and JSON storage.

```bash
# Navigate to the backend directory
cd backend

# Create a virtual environment (Recommended)
python -m venv venv

# Activate the virtual environment
# On Windows:
venv\Scripts\activate
# On MacOS/Linux:
source venv/bin/activate

# Install requirements (if you have a requirements.txt, otherwise skip or install fastapi/uvicorn manually)
pip install fastapi uvicorn pydantic

# Run the API in development mode
fastapi dev main.py
# The server will start running at http://127.0.0.1:8000
```

### 2️⃣ Setting up the Frontend
The frontend provides the interactive UI, graphs, and dashboards.

```bash
# Open a NEW terminal window and navigate to the frontend directory
cd frontend

# Install dependencies using pnpm
pnpm install

# Run the development server
pnpm run dev
# The application will start running usually at http://localhost:5173
```

## 🏗️ Architecture Overview

### Backend (Hexagonal Architecture)
The backend is strictly divided into layers to ensure high maintainability and testability:
- **`domain/`**: Contains core entities (`Student`, `Course`, `CurriculumGraph`) and the business logic (`AdvancedPlannerService` with the A* Engine).
- **`application/`**: Use cases connecting the REST layer to the domain logic.
- **`infrastructure/`**: Database repositories (currently using a robust JSON I/O implementation that reads and overwrites state perfectly) and external adapters.
- **`api/`**: The FastAPI controllers, routing, and schema definitions.

### Frontend
- **State Management**: Built on React Hooks with an emphasis on context-based propagation for the interactive nodes.
- **Visualization Tools**: Heavy lifting done by `reactflow` and `dagre` for the automatic layout calculation of the curriculum trees, and `recharts` for the dashboard analytics.

## 📝 Usage

1. Open your browser at `http://localhost:5173`.
2. Go to **"Mapa Curricular"** on the left sidebar to view your DAG.
3. Click on any grey node to view its details on the right pane, and click **"Marcar como Completada"** to pass the course. The UI will instantly update the backend's JSON and repaint the critical paths.
4. Go to **"Horario"** and click **"Auto-Generar"** to let the A* algorithm build an optimized schedule of up to 21 credits for your upcoming semester.
5. Visit the **"Dashboard"** to see your updated GPA, Credits pacing, and an AI-driven text insight summarizing your current academic momentum.

## ⚡ API Usage Examples (Swagger)

You can interact directly with the backend using the auto-generated Swagger UI at `http://127.0.0.1:8000/docs`. Here are some common operations you can perform:

### 1. Generating a Study Plan
**Endpoint:** `POST /students/{student_id}/plan`
*   Click on the endpoint to expand it.
*   Click the **"Try it out"** button.
*   **`student_id`**: Enter `00000000-0000-0000-0000-000000000001`
*   **Request Body**:
    ```json
    {
      "max_credits_per_semester": 21
    }
    ```
*   Click **Execute**. The engine will return a JSON list of semesters with the optimal courses to take.

### 2. Toggling a Course Status
**Endpoint:** `POST /students/{student_id}/history/toggle`
*   Click **"Try it out"**.
*   **`student_id`**: Enter `00000000-0000-0000-0000-000000000001`
*   **Request Body** (Use a valid course UUID from your graph):
    ```json
    {
      "course_id": "11111111-1111-1111-1111-111111111111"
    }
    ```
*   Click **Execute**. This will instantly modify the JSON database, either marking the course as passed or pending.

### 3. Checking Current GPA
**Endpoint:** `GET /students/{student_id}/gpa`
*   Click **"Try it out"**.
*   **`student_id`**: Enter `00000000-0000-0000-0000-000000000001`
*   Click **Execute**. You'll get your real-time calculated GPA based on the `curriculum.json` history.
